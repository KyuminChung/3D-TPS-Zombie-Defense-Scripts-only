# 3D-TPS-Zombie-Defense-Scripts-only
# [6주차 과제 기획서] 3D TPS 좀비 디펜스 프로토타입

## 1. 개요

- **장르:** 3D TPS (Third-Person Shooter) 디펜스
- **목표:** 3D 공간에서 몰려오는 적들을 처치하고 생존하는 것. (2D 로직의 3D화)
- **학습 목표**
    1. **3D 공간 이해:** `Vector3`, `Quaternion`(회전)의 기초 이해.
    2. **TPS 카메라 시스템:** 캐릭터를 따라다니는 3인칭 카메라 구현 (Cinemachine 활용 권장).
    3. **3D 애니메이션:** Mecanim 시스템, Animator Controller, Blend Tree (이동 모션 섞기).
    4. **AI 내비게이션:** `NavMeshAgent`를 이용한 길 찾기 (2D의 단순 벡터 추적을 대체).
    5. **3D Raycast:** 화면 중앙 (Crosshair)을 기준으로 한 사격 처리.

## 2. 핵심 게임 루프

1. **시작:** 플레이어는 3D 맵의 중앙 혹은 특정 위치에서 시작.
2. **전개:** 사방에서 적 (좀비)이 스폰되어 장애물을 피해 플레이어에게 접근.
3. **액션:** 플레이어는 WASD로 이동하고 마우스로 시점을 조절하며 적을 사격.
4. **결과:** 적을 처치하면 점수 획득, 플레이어 체력이 0이 되면 게임 오버.

## 3. 세부 기능 요구사항

### A. 플레이어

- **이동**
    - `Input.GetAxis`("Horizontal", "Vertical")를 사용하여 이동.
    - **필수:** 카메라가 바라보는 방향을 기준으로 앞/뒤/좌/우가 결정되어야 함 (월드 좌표 기준 이동 X).
- **회전**
    - 마우스 X축 움직임에 따라 캐릭터 (혹은 카메라)가 좌우로 회전.
- **애니메이션**
    - **Blend Tree 적용:** Idle ↔ Walk ↔ Run 모션이 입력값에 따라 부드럽게 섞이도록 구현.
    - 무료 3D 인간형 에셋 사용 (예: Unity Starter Assets 또는 Mixamo).

### B. 카메라

- **시점:** 숄더뷰 (어깨 너머로 캐릭터를 보는 시점).
- **구현 방식 (택 1)**
    - **옵션 1 (권장):** **Cinemachine** 패키지 설치 후 `Virtual Camera`를 사용하여 'Follow'(따라가기) 및 'LookAt' (바라보기) 설정.
    - **옵션 2 (하드코어할 수 있음):** 마우스 입력값을 받아 카메라의 Transform을 직접 제어하는 스크립트 작성.

### C. 사격 시스템

- **방식:** 2D의 투사체 방식 대신 **Raycast (히트스캔)** 방식 도입.
- **구현:**
    - 화면 정중앙 (Crosshair)에서 카메라가 바라보는 방향으로 `Physics.Raycast` 발사.
    - **충돌 처리:** 레이가 적 (`Enemy` 태그)에 닿으면 데미지 함수 호출 및 피격 이펙트 생성.
- **UI:** 화면 중앙에 조준점 (Crosshair) 이미지 고정.

### D. 적

- **이동 (NavMesh)**
    - 2D처럼 단순 직선 이동을 하면 벽에 막힘.
    - Unity **Navigation** 시스템을 베이크하고, 적에게 `NavMeshAgent` 컴포넌트 추가.
    - `agent.SetDestination(player.position)`을 통해 장애물을 피해 플레이어를 추적.
- **스폰**
    - 맵의 3~4군데 정해진 포인트에서 랜덤하게 생성.
    - **오브젝트 풀링:** 5주차에 배운 풀링 시스템을 3D 적에게도 적용 (선택 사항이지만 권장함).

## 4. 레벨 디자인 & 환경

- **맵**
    - 단순한 평지 (Plane) 위에 큐브 (Cube)로 벽과 장애물을 배치하여 미로 혹은 시가전 형태 구성.
    - 적들이 장애물을 피해 오는지 확인하기 위한 구조물 필수 배치.
- **라이팅**
    - Directional Light 1개 필수. (그림자 생성 확인)
